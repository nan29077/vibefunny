# VIBEACTION 테스트 시나리오

`admin@vibeaction.com / Admin1234!` 로 로그인하여 정책을 조정한 뒤, 별도 브라우저(또는 시크릿 창)에서 일반 계정으로 각 플로우를 검증합니다.

## A. 자동 검증 (이미 통과)
- `npm run build` — 35개 라우트 컴파일 성공.
- 핵심 ledger 로직 단위 시나리오 9/9 통과:
  - 영상 1회 판매 → `sold`, 재구매 차단
  - 판매 정산 = 가격 − 플랫폼수수료(20%)
  - 가입비 결제 → 회원 active + 추천 수당 = 가입비 × 추천율
  - 포인트 충전 잔액 반영

## B. 수동 시나리오

### 1. 무료 가입 (가입비 비활성, 기본)
1. `/signup` → VIBEACTION 회원으로 가입 → 즉시 `active`, 대시보드 진입.

### 2. 가입비 + 추천 수당
1. admin → `정책 설정` → VIBEACTION 회원 "가입비 사용" 체크, 금액 50000, 추천율 10% 저장.
2. 새 계정 가입 시 추천인 코드에 기존 회원 코드 입력 → 가입 후 결제 페이지.
3. "결제 성공" → 회원 `active`, 추천인의 `수익/출금`에 **지급 대기 5,000원** 생성.
4. admin `정산/출금` → "지급가능 처리" → 추천인 **지급 가능 5,000원**.

### 3. 영상 판매 (1회 한정)
1. creator `내 영상 → 등록`(저작권 체크 필수).
2. admin `영상 승인` → 승인 → 판매중.
3. buyer `영상 마켓` → 구매 → "결제 성공" → `구매 내역`에서 다운로드 가능.
4. 같은 영상은 마켓에서 사라지고(=sold), 재구매 불가.
5. creator `수익/출금`에 정산액(가격−20%) 반영.

### 4. 제작 의뢰
1. buyer `제작 의뢰 → 새 의뢰` + 예산 결제(성공).
2. creator `제작 의뢰`에서 참여 신청.
3. buyer `내 제작 의뢰`에서 작업자 선정.
4. creator 결과물 URL 제출 → buyer 승인 → creator 정산.

### 5. 광고 캠페인 + 실행사 수수료
1. advertiser(실행사) 가입 → `대행사/수수료`의 추천 코드 확인.
2. 다른 계정을 advertiser(대행사)로 가입하며 실행사 코드 입력 → 상위 실행사 연결.
3. 대행사 `포인트` 300,000 충전(성공).
4. 대행사 `캠페인 만들기` → 비용 자동 계산 → 포인트 차감 → 관리자 승인 대기.
5. admin `캠페인 승인/노출` → 승인.
6. creator `광고 캠페인` → 참여 → (admin/광고주 승인) → 증빙 제출.
7. admin/광고주 증빙 승인 → creator 수익 정산.
8. 실행사 `대행사/수수료`에 수수료(기본: 대행사 차감액 × 5%) 생성 확인.

### 6. 출금
1. creator `수익/출금` → 출금 신청.
2. admin `정산/출금` → 승인 → 지급완료.

### 7. 반응형
- 브라우저 폭을 좁히면 사이드바 → 상단 햄버거 + 하단 탭 메뉴로 전환.
